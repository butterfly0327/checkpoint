# README — 공공데이터포털(data.go.kr) 메타데이터 수집 (API)

## 목표 (2차 MVP 범위)
- data.go.kr 내 “개방 목록/오픈API 목록”에 대한 카탈로그 메타(M1~M13)를 수집한다.
- 개별 원천 데이터 파일은 다운로드/분석하지 않는다.
- 단, 포털이 제공하는 “목록개방현황(메타데이터)” 파일/오픈API는 메타 수집 대상으로 사용한다(실데이터가 아님).

## 수집 방식 (선택): API
- 가장 안정적인 1차 수집원:
  - 공공데이터활용지원센터_목록조회서비스(오픈API)
  - 공공데이터활용지원센터_공공데이터포털 목록개방현황(파일데이터/오픈API 변환 제공)

## 수집 대상 (내부 스키마)
- M1: source="datagokr", source_id(목록ID/목록URL), canonical_url
- M2: 목록명(데이터명/오픈API명)
- M3: 설명/기타 유의사항/데이터 한계
- M4: 분류체계(카테고리)
- M5: 제공유형(파일/API/표준), 확장자, 제공형태
- M6: 이용허락범위(제한 없음/조건 등)
- M7: 제공기관/관리부서
- M8: 키워드
- M9: 등록일/수정일/업데이트 주기/차기 등록 예정일(있으면)
- M10: (제한) 사이트가 스키마를 표준으로 제공하지 않는 경우 기본 비움
- M11: 전체 행(제공 시), 포맷/용량(제공 시)
- M12: 신청가능 트래픽, 심의유형, 활용신청 필요 여부
- M13: 조회수(제공 시)

## API 사용 준비
- data.go.kr 서비스키(ServiceKey) 발급 필요
- 일부 오픈API는 “활용신청”이 필요할 수 있음(메타로 수집)

## 수집 파이프라인(권장)
1) “목록개방현황”을 월 1회 풀스캔(포털 전체 목록을 한 번에 확보)
2) “목록조회서비스”를 매일/매시간 증분 체크(등록/수정된 목록 탐지)
3) 상위 N개(조회수/최근 수정)만 상세 페이지 메타(DCAT/schema.org 링크 제공 시) 추가 수집

## 증분(이어 긁기) 설계
- upsert_key = (source, source_id)
- checkpoint = (수정일 max, 파일 버전/등록일)
- 파일데이터가 월간 갱신이면: “최신 파일명(YYYYMMDD)” 기준으로 새 버전만 수집

## PoC 성공 기준(최소)
- 목록개방현황 1개 버전에서 5만 건 이상 메타 수집 가능 여부 확인
- 레코드당 M1~M9, M12는 90% 이상 채움
- 조회수/트래픽/심의유형까지 M13/M12로 저장
