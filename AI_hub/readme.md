# README — AI Hub 메타데이터 수집 (크롤링)

## 목표 (2차 MVP 범위)
- AI Hub “데이터셋 상세 페이지”에 노출된 메타정보(M1~M13 중 제공되는 범위)를 수집한다.
- 데이터 파일(원천 데이터) 다운로드/분석은 하지 않는다.
- 즉, 사이트에 이미 공개된 “메타데이터 구조표/데이터 통계/소개/목적/변경이력”만 파싱한다.

## 수집 방식 (선택): 크롤링(HTML)
- 이유: 공개적으로 문서화된 “카탈로그 API”가 명확하지 않으며,
  상세페이지에 이미 핵심 메타가 구조적으로 존재한다.

## 대상 URL 패턴(상세 페이지)
- https://www.aihub.or.kr/aihubdata/data/view.do?...&dataSetSn={DATASET_SN}...

PoC에서는 우선 dataSetSn 목록을:
1) 사이트 목록 페이지에서 수집하거나
2) 사이트맵(sitemap.xml)이 제공되면 거기서 수집하거나
3) 내부 seed 리스트(수작업 200개)로 시작한다.

## 파싱 대상 섹션(예시)
- 소개 / 구축목적 (M3)
- 메타데이터 구조표: 데이터 영역, 데이터 유형, 데이터 형식, 데이터 출처, 라벨링 유형, 라벨링 형식, 구축년도/구축량 (M4/M5/M10/M11)
- 데이터 통계 표 (M11)
- 데이터 변경이력/히스토리 (M9)

## 수집 대상(M1~M13)
- M1: source="aihub", source_id=dataSetSn, canonical_url
- M2: 데이터셋 제목
- M3: 소개/설명/구축목적 텍스트
- M4: 데이터 영역/카테고리(텍스트 기반 분류)
- M5: 데이터 형식(예: txt), 라벨링 형식(예: JSON)
- M6: (제한) 데이터셋별 라이선스가 명시되면 수집, 없으면 비움
- M7: (제공 시) 구축기관/참여기관/담당(페이지에 있으면)
- M8: (제공 시) 키워드/태그
- M9: 변경이력/버전/일자
- M10: 라벨링 유형/형식(사이트에 적힌 값)
- M11: 구축량/통계 수치(페이지에 적힌 값)
- M12: 접근 제약(“다운로드 절차/승인 필요” 같은 정책 문구는 ‘정책 요약’으로만 저장)
- M13: (제공 시) 조회수/활용신청 수치, 없으면 비움

## 크롤링 제약/컴플라이언스 (중요)
- robots.txt / 이용약관을 준수한다.
- 로그인/캡차/승인 과정을 “우회”하지 않는다.
- 과도 요청을 피하기 위해:
  - 요청 간 딜레이(예: 0.5~2초 랜덤)
  - 동시성 제한(예: 2~4)
  - 429/5xx 발생 시 지수 백오프
  - User-Agent 명시 + 연락 가능한 메일/서비스명 표기(선택)

## 증분(이어 긁기)
- upsert_key=(source, dataSetSn)
- checkpoint 전략 2가지(둘 중 하나 선택)
  1) 페이지 내 “데이터 변경이력/히스토리”의 최신 일자를 저장 → 이후 최신 일자보다 갱신된 페이지만 재수집
  2) HTTP 캐시(ETag/Last-Modified)가 제공되면 If-None-Match/If-Modified-Since로 304 활용

## PoC 성공 기준
- dataSetSn 500개를 안정적으로 순회하며
  - M1~M5, M9~M11을 80% 이상 채움
  - 파싱 실패율 < 3%
- HTML 구조 변경에 대비해 파서 로직을 “CSS selector + fallback(텍스트 탐색)” 2단으로 구성
